# 📚 Git Шпаргалка для студентов

## 🎯 Что такое Git? (Очень просто)

**Git** — это программа, которая **запоминает все версии твоих файлов**.

Представь, что ты делаешь "фотографии" своей папки с проектом. В любой момент можно вернуться к любой фотографии.

| Термин | Объяснение простыми словами |
|--------|----------------------------|
| **Git** | Программа на твоём компьютере |
| **GitHub** | Сайт, где хранится копия проекта |
| **Коммит** | "Фотография" твоих файлов в этот момент |
| **Ветка** | "Черновик" для экспериментов |
| **Main** | Главная, чистая версия проекта |

---

## ⚡ Главные команды (топ-5)

```bash
git status      # 🔍 Что изменилось?
git add .       # ➕ Добавить всё
git commit -m "описание"  # 📸 Сделать снимок
git push        # ☁️ Отправить на GitHub
git pull        # 📥 Забрать с GitHub

🌿 Ветки (Branch) — твои черновики
Ветка — это как черновик в Word. Ты можешь экспериментировать, ничего не ломая в главной версии.

bash
git branch                    # 📋 Посмотреть все ветки
git checkout -b имя-ветки     # ✨ Создать новую ветку
git checkout имя-ветки        # 🔄 Переключиться на ветку
git push -u origin имя-ветки  # ☁️ Отправить ветку на GitHub
Простое правило:

main = чистовик (всё работает)

feature/что-то = черновик (экспериментирую)

🚀 Как начать новый проект (пошагово)
bash
# 1. Создать папку
mkdir мой-проект
cd мой-проект

# 2. Создать Git
git init

# 3. Создать виртуальное окружение (Python)
python -m venv .venv
source .venv/Scripts/activate  # на Windows

# 4. Добавить файлы и сохранить
git add .
git commit -m "Первый коммит"

# 5. Связать с GitHub (сначала создать репозиторий НА САЙТЕ)
git remote add origin https://github.com/твой-логин/название.git
git push -u origin main

🧭 Весь рабочий день с Git (шпаргалка)
bash
# УТРО — начать работу
git checkout main           # перейти в главную ветку
git pull origin main        # забрать свежие изменения

# СОЗДАТЬ ВЕТКУ ДЛЯ НОВОЙ ФИЧИ
git checkout -b feature/моя-фича

# РАБОТАТЬ — писать код, создавать файлы

# СОХРАНИТЬ ПРОМЕЖУТОЧНЫЙ РЕЗУЛЬТАТ
git add .
git commit -m "частично сделала"

# ВЕЧЕР — закончить и отправить
git add .
git commit -m "готовая фича"
git push -u origin feature/моя-фича

# НА САЙТЕ GITHUB: создать Pull Request → нажать Merge

# Обновить главную ветку
git checkout main
git pull origin main

💡 Золотые правила Git
Правило	Почему это важно
Перед созданием ветки — git status	Чтобы всё было чисто
Понятные коммиты	Через месяц ты поймёшь, что делала
Не работай прямо в main	Ветки — для экспериментов
Чаще пуши	Чтобы не потерять код
Не бойся ошибок	Всё можно отменить
🔧 Полезные команды (запасные)
bash
git log --oneline                     # История коммитов (кратко)
git branch -d имя-ветки               # Удалить ветку локально
git checkout main -- файл.py          # Восстановить файл из main
git reset --hard хеш-коммита          # Откатиться к старой версии
git revert хеш-коммита                # Безопасно отменить коммит
git remote -v                         # Посмотреть связь с GitHub
🧠 САМОЕ ВАЖНОЕ (запомни навсегда)
Git не удаляет файлы насовсем. Всё, что было закоммичено, можно вернуть.

Файлы "пропадают" при переключении веток, потому что их там нет. Они есть только в тех ветках, где были закоммичены.

Всего 5 команд решают 90% задач: status, add ., commit, push, pull
